# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive problems.

Until a dedicated security contact is added, treat this repository as pre-release and avoid publishing secrets, tokens, private datasets, or internal documents in issues and pull requests.

## Scope

Security-sensitive areas for this project will include:

- source ingestion and document parsing
- prompt injection resistance for source-grounded workflows
- artifact generation and file handling
- any future web or API surfaces
